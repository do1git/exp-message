# 패치노트

이 디렉토리는 프로젝트의 데일리 패치노트를 기록하는 공간입니다.

## 디렉토리 구조

패치노트는 주차별로 폴더에 정리됩니다.

```
patch/
  week-2026-01-05/          # 주차별 폴더 (월요일 날짜)
    README.md               # 주차별 요약
    day2-01-*.md            # 일별 패치노트
    day3-01-*.md
    ...
  week-2026-01-12/          # 다음 주차
    ...
```

### 주차별 폴더 명명 규칙

- 형식: `week-YYYY-MM-DD` (해당 주의 월요일 날짜)
- 예: `week-2026-01-05` (2026년 1월 5일이 월요일인 주)

## 작성 방법

각 날짜별로 패치노트를 작성합니다. 버전이 추가된 날에는 파일명에 버전을 함께 표기합니다.

### 파일 명명 규칙

주차별 폴더 내부의 패치노트 파일은 다음과 같은 형식을 사용합니다:

- 기본 형식: `dayN-번호-요약.md` (예: `day2-01-프로젝트초기설정.md`, `day3-02-ApiResponseTemplate구현.md`)
  - `dayN`: 주차 내 일차 번호 (day1, day2, day3, ...)
  - `번호`: 같은 날 여러 패치노트를 작성할 경우 증가 (01, 02, ...)
  - `요약`: 간단한 설명
- 버전 추가 형식: `dayN-번호-Release-v0.0.1.md` (예: `day2-01-Release-v0.0.1.md`)

**일차 번호:**
- 주차의 월요일부터 시작하여 day1, day2, day3, ... 순서로 증가
- 예: 월요일=day1, 화요일=day2, 수요일=day3, ...

같은 날 여러 패치노트를 작성할 경우 번호를 증가시켜 구분합니다. 파일명에 간단한 설명을 포함하여 내용을 쉽게 파악할 수 있도록 합니다. 버전이 릴리즈된 날에만 버전을 파일명 뒤에 추가합니다.

### 주차별 요약 파일

각 주차 폴더에는 `README.md` 파일을 생성하여 해당 주차의 주요 작업 내용을 요약합니다.

### 패치노트 템플릿

```markdown
# 패치노트 - YYYY-MM-DD

## 목표했던 내용 1
- 

## 변경사항 1
- 

## 목표했던 내용 2
- 

## 변경사항 2
- 
```

각 목표와 그에 대응하는 변경사항을 1:1로 매핑하여 인과관계를 명확히 합니다.
